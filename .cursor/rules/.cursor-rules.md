# Project Rules – ReelForge MVP

## General Principles
- API-first architecture
- Backend contains all business logic
- Frontend is a thin client
- Code must be modular and scalable
- Follow clean architecture principles

## Backend (Laravel)

- Use RESTful API structure
- Use FormRequest for validation
- Use Service layer for video generation logic
- Use Jobs for video processing
- Never generate video synchronously
- Use DTO for data transfer
- Follow PSR standards
- Avoid fat controllers

### Structure
- Controllers only handle request/response
- Business logic in Services
- Heavy operations in Jobs
- Use Redis for queues

## Frontend (React)

- Use functional components
- Use hooks only
- Separate UI and API logic
- Create reusable components
- Keep components small

## API Rules

- All routes must be under /api
- JSON responses only
- Proper HTTP status codes
- Standard error structure:
  {
    "success": false,
    "message": "",
    "errors": []
  }

## Video System

- All videos must be vertical 9:16
- Max duration: 15 seconds
- Resolution: 1080x1920
- Use ffmpeg
- Store output in cloud storage

## Security

- Use Sanctum tokens
- Validate all inputs
- Limit file size uploads
- Allow only images (jpg, png, webp)

## Future Ready

- Mobile-ready API
- Subscription-ready structure
- Scalable queue system
